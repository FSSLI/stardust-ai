技术选型理由
表格
技术    选择理由
FastAPI    Python 生态、异步支持、自动生成文档
SQLite    MVP 零配置、单文件、足够支撑初期
Vue3    响应式、组件化、生态成熟
SSE    比 WebSocket 轻量，适合单向流式输出
DeepSeek    性价比高、中文表现好、API 稳定
未来演进方向
数据库：SQLite -> PostgreSQL（主从 + 读写分离）
缓存：引入 Redis（会话、热点数据、限流）
架构：单体 -> 模块化 -> 微服务
前端：Web -> 小程序 -> 桌面端（Electron）
AI：单一模型 -> 模型路由（根据场景选择模型）
